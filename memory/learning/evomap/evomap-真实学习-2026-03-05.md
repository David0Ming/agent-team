## 真正学到的知识

### 1. 飞书消息发送最佳实践 (GDI 59.70, 273 calls) 🔥

**问题**: 飞书消息发送失败的各种情况
- feishuformaterror - 格式错误
- markdown_render_failed - markdown渲染失败
- card_send_rejected - 卡片发送被拒绝
- feishu_api_timeout - API超时

**解决方案**:
- 自动重试机制 (auto_retry)
- 降级策略：富文本 → 纯文本 → 失败通知
- circuit_breaker_open - 熔断机制

### 3. 飞书 @提及通知 (GDI 44.65)

**问题**:
- feishunotificationfailure - 通知失败
- userdidnotseecard - 用户没看到卡片
- mentionnotworking - @提及不工作
- cardpushfailure - 卡片推送失败

### 4. 核心知识点

**4.1 消息发送降级策略**:
```
富文本消息 → 失败 → 纯文本消息 → 失败 → 邮件/短信 fallback
```

**4.2 熔断机制**:
- 失败次数超过阈值 → 打开熔断
- 熔断期间快速失败
- 冷却期后尝试恢复

**4.3 指数退避 + Jitter**:
- 重试间隔：1s, 2s, 4s, 8s...
- 加上随机 jitter 避免惊群效应

### 5. 实际应用

我们目前的飞书工具可以改进：
1. 添加重试机制 (当前没有)
2. 添加降级策略
3. 添加错误日志记录
