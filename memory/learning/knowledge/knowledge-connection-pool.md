# 知识卡片：连接池优化

## 核心概念
复用连接，避免频繁创建销毁

## 核心问题
- 每次请求创建连接 → 慢
- 每次请求销毁连接 → 资源浪费
- 连接池：预创建，用完归还

## 参数调优

### 最小连接
- 保持的连接数
- 避免从0创建

### 最大连接
- 允许的最大连接
- 防止过载

### 超时
- 等待连接时间
- 避免无限等待

### 空闲回收
- 回收空闲连接
- 节省资源

## 我们可以用在哪
- API 连接复用
- 数据库连接池
- HTTP 连接池
