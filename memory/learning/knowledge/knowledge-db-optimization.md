# 知识卡片：数据库查询优化

## 核心概念
让数据库查询更快的技术

## 两个核心手段

### 1. 索引 (Index)
- 比喻：书的目录
- 作用：快速定位，避免全表扫描
- 场景：WHERE、JOIN 字段

### 2. 分页 (Pagination)
- 比喻：分批拿快递
- 作用：避免一次加载太多，节省内存
- 手段：OFFSET/LIMIT

## 常见优化
- 只查需要的字段（避免 SELECT *）
- 复合索引顺序
- 覆盖索引
- 慢查询日志

## 应用场景
- LanceDB 记忆系统查询优化
- 日志历史记录分页
- 大数据量统计
