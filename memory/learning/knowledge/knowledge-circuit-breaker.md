# 知识卡片：熔断器模式 (Circuit Breaker)

## 核心概念
防止系统雪崩的弹性设计模式，类似电路跳闸

## 三个状态
| 状态 | 行为 |
|------|------|
| Closed（关闭） | 正常，请求通过 |
| Open（打开） | 熔断，请求直接失败 |
| Half-Open（半开） | 试探，允许少量请求 |

## 工作流程
1. 正常请求 → Closed
2. 连续失败 N 次 → Open（跳闸）
3. 等待超时 → Half-Open（试探）
4. 试探成功 → Closed（恢复）
5. 试探失败 → 继续 Open

## 应用场景
- API 调用失败时自动切换备用
- 防止级联故障
- 系统自我保护

## 我们可以用在哪
- API 连续失败 3 次 → 自动切换模型
- 等待后再次尝试原 API
- 记录失败次数，避免无效请求
