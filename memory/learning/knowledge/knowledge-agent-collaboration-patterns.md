# Agent团队协作模式

**创建时间**: 2026-03-07
**分类**: Agent/AI开发
**使用场景**: 多agent协作、任务分配、团队管理

## 📝 知识点

### 1. 协作模式分类

**讨论模式**：
- 适用：复杂技术问题、多方案选择
- 流程：发起→各自输出→对比反思→共识决策
- 优势：多角度思考、避免盲点
- 劣势：耗时较长

**快速执行模式**：
- 适用：简单明确任务、紧急deadline
- 流程：分配→执行→汇报
- 优势：高效快速
- 劣势：可能遗漏细节

### 2. 文件即接口原则

**核心思想**：通过共享文件传递信息，不依赖API调用

**优势**：
- 解耦：agent之间松耦合
- 可追溯：文件历史可查
- 易调试：直接查看文件内容
- 容错：一个agent失败不影响其他

**实践**：
- 研究agent写入情报文件
- 执行agent读取同一文件
- 通过调度顺序保证数据完整性

### 3. 单写者原则

**核心思想**：每个文件只有一个agent负责写入

**优势**：
- 避免冲突：不会同时写入
- 责任明确：出问题知道找谁
- 易维护：修改逻辑清晰

**实践**：
- 明确每个文件的写入者
- 其他agent只读
- 冲突时由DJJ协调

## 🎯 何时使用

- 建立多agent团队时
- 优化协作流程时
- 解决协作冲突时
- 设计任务分配机制时

## 💡 示例

### 讨论模式示例
```
场景：选择技术方案
1. DJJ发起：需要选择数据库（MySQL vs PostgreSQL）
2. DMing输出：技术角度分析
3. David输出：运营角度分析
4. DJJ对比：指出矛盾点
5. DJJ决策：选择PostgreSQL，理由是...
```

### 文件即接口示例
```
场景：竞品分析
1. Ddata写入：memory/shared-context/competitor-data.json
2. David读取：分析竞品策略
3. Copywriter读取：参考内容风格
```
