# 知识卡片：健壮 HTTP 处理

## 核心概念
让 HTTP 请求在不可靠网络中可靠运行

## 四个核心技术

### 1. 超时控制
- 设置合理请求超时
- 避免无限等待

### 2. 重试策略
- 失败后重试
- 指数退避：等待时间翻倍，避免雪崩

### 3. 幂等设计
- 重复请求不产生副作用
- GET/DELETE 天然幂等
- POST/PUT 需要设计

### 4. 熔断器
- 连续失败后暂停
- 恢复后继续

## 应用场景
- API 请求可靠性
- 网络不稳定环境
- 高并发保护

## 我们可以用在哪
- Fox/Kimi API 超时控制
- 失败重试策略
- 连续失败熔断
