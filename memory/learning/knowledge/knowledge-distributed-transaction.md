# 知识卡片：分布式事务

## 核心概念
跨多个服务/数据库的操作，要么全成功，要么全失败

## 三种实现方案

### 1. 2PC（两阶段提交）
- 准备阶段：各服务检查能否执行
- 提交阶段：全部提交或全部回滚

### 2. TCC（Try-Confirm-Cancel）
- Try：预留资源
- Confirm：确认执行
- Cancel：取消预留

### 3. 消息队列
- 最终一致性
- 通过消息保证最终成功

## 应用场景
- 多模型切换回滚
- 跨Agent操作原子性
- 配置文件更新

## 我们可以用在哪
- API 失败时的自动回滚
- 多步骤操作的原子性
