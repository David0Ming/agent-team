## 题目1：二维数组中的查找

### 问题描述
在行列都递增的二维数组中查找目标值

### 你的初始思路
- 沿对角线搜索
- 假设：矩阵是正方形
- 逻辑：左上方块<当前元素<右下方块

### 遇到的问题
1. **数组越界**：j会超出列数范围（因为假设了正方形）
2. **段错误**：空矩阵`[[]]`导致`array[0].size()`访问失败
3. **通过率**：13/21（非正方形矩阵会漏掉元素）

### 你问的问题
1. "为什么正确率不是百分百？"
2. "我的思路是左上方块小于当前元素，右下方块大于它"
3. "你解释一下为什么会漏掉元素吧"

### 核心错误
- **假设矩阵是正方形** → 非正方形时会漏掉对角线外的元素
- **while循环重复搜索** → 同样的范围搜索多次，没有意义

### 你的思考过程
**关于while循环重复搜索**：
- 最初：不理解为什么是"重复搜索"
- 思考后：意识到"因为每行每列都是单调的"
- 顿悟：如果第一次在某个范围内没找到，后面再搜索同样的范围也不会找到
- 结论：while循环的k只是计数器，没有改变搜索范围，所以是无意义的重复

### 正确思路
**从右上角（或左下角）开始搜索**：
- 右上角：该行最大，该列最小
- 当前值 > target → 向左移动
- 当前值 < target → 向下移动
- 每次移动排除一行或一列

### 正确代码
```cpp
bool Find(int target, vector<vector<int>>& array) {
    if (array.empty() || array[0].empty()) return false;
    
    int i = 0, j = array[0].size() - 1;  // 右上角
    
    while (i < array.size() && j >= 0) {
        if (array[i][j] == target) return true;
        else if (array[i][j] > target) j--;  // 向左
        else i++;  // 向下
    }
    return false;
}
```

### 关键收获
1. **不要假设数据结构**：矩阵不一定是正方形
2. **利用有序特性**：找到"既能增大又能减小"的起点
3. **处理边界情况**：空矩阵、空数组
4. **时间复杂度**：O(n+m)，空间复杂度O(1)

## 题目2：重建二叉树

### 问题描述
根据前序和中序遍历重建二叉树

### 你的疑问
1. "vinMap是什么？vinOrder是什么？题目说了吗？"
2. "为什么要给他们赋值i？"
3. "为什么要创造这样的一个build函数？"
4. "我知道怎么分解但是我不知道怎么把所有左右子树连接到根节点"

### 核心思路
1. **前序第一个元素 = 根节点**
2. **在中序中找根节点位置 → 划分左右子树**
3. **递归构建左右子树**
4. **连接左右子树到根节点**

### 关键理解

**vinMap的作用**：
- 哈希表：值 → 索引
- 优化查找：O(n²) → O(n)
- 不是题目给的，是优化手段

**build函数的作用**：
- 递归需要传递范围参数（起始、结束位置）
- 题目函数签名固定，无法传递额外参数
- build函数返回：构建好的子树的根节点

**连接是自动的**：
```cpp
root->left = build(...);   // 递归返回左子树根，赋值给left
root->right = build(...);  // 递归返回右子树根，赋值给right
```

### 正确代码
```cpp
class Solution {
public:
    TreeNode* reConstructBinaryTree(vector<int>& pre, vector<int>& vin) {
        unordered_map<int, int> vinMap;
        for (int i = 0; i < vin.size(); i++) {
            vinMap[vin[i]] = i;  // 值 → 索引
        }
        return build(pre, 0, pre.size()-1, vin, 0, vin.size()-1, vinMap);
    }
    
    TreeNode* build(vector<int>& pre, int ps, int pe,
                    vector<int>& vin, int vs, int ve,
                    unordered_map<int, int>& vinMap) {
        if (ps > pe) return nullptr;
        
        TreeNode* root = new TreeNode(pre[ps]);  // 根节点
        int rootIdx = vinMap[pre[ps]];           // 在中序找位置
        int leftSize = rootIdx - vs;             // 左子树大小
        
        root->left = build(pre, ps+1, ps+leftSize, vin, vs, rootIdx-1, vinMap);
        root->right = build(pre, ps+leftSize+1, pe, vin, rootIdx+1, ve, vinMap);
        
        return root;
    }
};
```

### 关键收获
1. **递归的精髓**：相信递归函数会返回正确结果，直接使用
2. **哈希表优化**：空间换时间，O(n²) → O(n)
3. **范围参数**：递归需要明确处理的子数组范围
4. **连接是自动的**：`root->left = build(...)`一行完成递归+连接

## 通用经验

### 解题方法论
1. **不要假设，要验证**
   - 错误示例：假设矩阵是正方形
   - 正确做法：检查rows和cols，用min(rows, cols)
   - 教训：任何假设都要在代码中验证

2. **利用题目给的特性**
   - 题目说"递增"就一定要用这个特性
   - 找到能同时"增大"和"减小"的起点（如右上角）
   - 不要暴力遍历，要利用单调性剪枝

3. **边界情况必须处理**
   - 空数组：`if (array.empty())`
   - 空矩阵：`if (array[0].empty())`
   - 单元素：也要能正确处理
   - 先写边界判断，再写主逻辑

4. **优化要有理由**
   - 哈希表：用空间换时间（O(n²) → O(n)）
   - 双指针：利用单调性减少遍历
   - 不要为了优化而优化，要理解为什么

### 递归的正确理解
1. **三要素**
   - 返回值：这个函数返回什么？（如：子树的根节点）
   - 终止条件：什么时候停？（如：ps > pe）
   - 递归调用：如何缩小问题？（如：划分左右子树）

2. **相信递归**
   - 不要展开递归过程（会绕晕）
   - 相信递归会返回正确结果
   - 直接使用返回值（如：root->left = build(...)）

3. **辅助函数的作用**
   - 题目函数签名固定，无法传递额外参数
   - 递归需要范围参数（起始、结束位置）
   - 所以创建辅助函数（如：build函数）

### 调试技巧
1. **看报错信息**
   - 段错误 → 数组越界或空指针
   - 通过率X/Y → 找没通过的用例特点
   - 超时 → 复杂度太高，需要优化

2. **手动模拟小例子**
   - 用2x2或3x3的小矩阵测试
   - 用3个节点的小树测试
   - 走一遍流程，看哪里出错

3. **打印关键变量**
   - 递归层次、当前范围
   - 中间结果、返回值
   - 但提交前要删除打印语句

### 思考的重要性
**不要只记答案，要记思考过程**：
- 为什么最初的思路错了？
- 什么时候顿悟的？
- 正确思路的关键点是什么？
- 如果再遇到类似题目，能想到这个方法吗？

**示例（今天的收获）**：
- while循环重复搜索：最初不理解，思考"单调性"后顿悟
- 递归连接：最初不理解，理解"返回值"后顿悟
- 这些顿悟时刻比答案本身更重要
