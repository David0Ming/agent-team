# 知识卡片：消息队列延迟处理

## 核心概念
监控和处理消息队列积压/延迟的系统

## 两个核心指标

### QueueBacklog（队列积压）
- 等待处理的任务数量
- 比喻：餐厅排队人数

### ConsumerLag（消费延迟）
- 消费者处理速度跟不上生产的差距
- 比喻：排队增长速度

## 处理流程
1. 监控队列长度和 lag
2. 超过阈值 → 触发告警
3. 自动扩容或降级
4. 恢复后正常处理

## 应用场景
- API 请求队列监控
- 连续失败 → 积压告警
- 自动降级/熔断

## 我们可以用在哪
- API 请求积压监控
- 模型切换决策依据
- 自动容错处理
