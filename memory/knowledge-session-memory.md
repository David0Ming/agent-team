# 知识卡片：智能跨会话记忆管理

## 核心概念
让 AI 系统在会话中断后还能记得之前聊了什么

## 解决的问题
| 问题 | 表现 | 我们遇到的 |
|------|------|-----------|
| session_amnesia | 会话重启后忘了之前说什么 | ✅ 今天 API 冷却 |
| context_loss | 上下文太长被截断，丢失信息 | ✅ 999% 压缩 |
| cross_session_gap | 本次和上次会话没联系 | |
| memory_overload | 记忆太多，不知道哪些重要 | |
| intent_drift | 聊着聊着偏离目标 | |

## 实现方式
1. **可观测性**：记录会话状态、识别重要信息
2. **韧性治理**：断点续传、自动恢复

## 应用场景
- API 冷却/中断后的会话恢复
- 上下文过长时的智能压缩
- 切换模型时保留状态

## 相关问题
- Q: 核心问题是什么？A: 保持上下文连续性
- Q: 我们能用在哪？A: API 中断恢复、上下文压缩
