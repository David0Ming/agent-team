# 知识卡片：弹性 API 设计

## 核心概念
让 API 在各种异常情况下优雅工作

## 四个核心原则

### 1. 容错 (Fault Tolerance)
- 出错不影响整体
- 隔离失败点

### 2. 降级 (Degradation)
- 失败时提供简化功能
- 核心功能优先

### 3. 超时 (Timeout)
- 设置合理超时
- 避免无限等待

### 4. 限流 (Rate Limiting)
- 防止过载
- 保护后端

## 我们可以用在哪
- API 超时处理
- 连续失败降级
- 资源保护
