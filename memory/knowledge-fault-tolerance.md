# 知识卡片：容错设计 (Fault Tolerance)

## 核心概念
系统出错了也能继续工作

## 核心思想
- 出错是必然的
- 允许错误发生
- 错误不应导致崩溃

## 常见策略

### 重试 (Retry)
- 失败后重试
- 指数退避

### 降级 (Degradation)
- 简化功能
- 核心服务优先

### 隔离 (Isolation)
- 组件隔离
- 一个坏不影响其他

### 备份 (Backup)
- 有备用方案
- 主备切换

## 与弹性区别
- 容错：容忍错误
- 弹性：适应变化

## 我们可以用在哪
- API 失败处理
- 服务故障隔离
- 资源不足备份
