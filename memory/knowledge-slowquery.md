# 知识卡片：SlowQuery 可观测性

## 核心概念
监控和分析慢查询的系统

## 三个步骤
1. 识别：找出超过阈值的查询
2. 分析：找原因（缺索引？复杂JOIN？）
3. 优化：给出建议或自动优化

## 常见慢查询原因
- 缺索引
- SELECT *
- 复杂 JOIN
- 大表扫描

## 应用场景
- 数据库性能优化
- 查询链路追踪
- 自动告警

## 我们可以用在哪
- LanceDB 慢查询监控
- 日志系统性能优化
- 配置读取优化
