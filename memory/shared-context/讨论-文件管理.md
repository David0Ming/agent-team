# 讨论：文件管理和记忆机制优化

## 背景

我们有大量md文件和复杂的记忆机制，但存在"文件越来越多 却不知道何时读哪个"的问题。

## 当前文件体系

| 文件 | 作用 |
|------|------|
| TEAM.md | 核心清单 |
| HEARTBEAT.md | 心跳任务 |
| AGENTS.md | 代理规则 |
| SOUL.md | 角色设定 |
| USER.md | 用户信息 |
| memory/projects.md | 待办任务 |
| memory/knowledge-*.md | 学到的知识 |
| memory/YYYY-MM-DD.md | 每日记录 |

## 问题

- 文件分散在不同位置
- 没有自动索引机制
- 需要人工记住"何时读哪个"

## 方案对比

### 方案A：在TEAM.md加"快速索引"表格

**优点**：简单，一页纸看清
**缺点**：仍需人工查阅

### 方案B：单一入口文件

把核心内容整合到一个文件，按section区分
**优点**：不分散
**缺点**：文件会越来越长

### 方案C：自动记忆召回

用memory_recall工具，需要时召回
**优点**：智能
**缺点**：依赖向量搜索，不够精准

### 方案D：保持现状+优化 TEAM.md
