# 知识卡片：事件驱动架构

## 核心概念
基于事件触发的系统设计

## vs 请求驱动
- 请求驱动：请求 → 处理 → 响应
- 事件驱动：事件 → 触发 → 反应

## 核心组件

### 事件
- 发生的事情
- 状态变化

### 事件源
- 产生事件的地方
- 如：用户点击、API调用

### 事件总线
- 传递事件的管道
- 解耦生产者和消费者

### 事件处理器
- 处理事件的逻辑
- 响应事件

## 优势
- 解耦
- 可扩展
- 异步

## 我们可以用在哪
- 事件触发动作
- 解耦模块
- 异步处理
