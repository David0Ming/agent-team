# 知识卡片：异步批处理

## 核心概念
将任务批量处理，提高效率

## vs 同步处理
- 同步：1个请求 = 1次处理
- 批处理：N个请求 = 1批处理

## 优势

### 减少开销
- 批量操作更高效
- 减少重复工作

### 减少IO
- 合并网络请求
- 减少数据库连接

### 提高吞吐
- 整体处理更快
- 资源利用率高

## 应用场景
- 日志收集
- 数据统计
- 邮件发送

## 我们可以用在哪
- 批量日志处理
- 批量 API 请求
- 批量状态更新
