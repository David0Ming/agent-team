# DJJ个人经验库

## 内容生产经验

### 敏感词汇管理（2026-03-10）
**场景**：嘟嘟巴士prompt生成时发现使用了"暴力"、"欺骗"、"品牌"等敏感词汇

**原则**：
1. 任何面向公众的内容，必须避免负面情绪词（暴力、欺骗、恐惧、焦虑）
2. 避免明显的商业化词汇（品牌、广告、营销、推广）
3. 建立敏感词替代词库，保持创意的同时确保合规

**替代策略**：
- 负面词 → 中性描述（"暴力" → "对比"）
- 商业词 → 具体名称（"品牌" → "嘟嘟巴士"）

### 数据分析方法（2026-03-10）
**场景**：竞品数据分析时，用户要求"昨天单日数据"而不是累计数据

**原则**：
1. 明确数据类型：单日增量 vs 累计统计
2. 单日增量 = 今天数据 - 昨天数据
3. 分析必须多维度：总体数据、TOP表现、平台对比、策略洞察、行动建议

**价值**：
- 单日数据更能反映真实趋势
- 避免被累计数据误导
- 便于快速响应市场变化

### 合规优先原则（2026-03-10）
**场景**：生成内容后多次修改敏感词汇

**教训**：应该在生成前就建立合规检查清单，而不是生成后再修改

**改进**：
1. 建立项目级合规清单（存储在项目docs/）
2. 生成内容前先过一遍清单
3. 使用自动化检查（grep敏感词）
