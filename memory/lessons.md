## 技术教训

### 知识管理的三个层次（2026-03-06）
**问题**：学到53个知识卡片，但不会在需要时想起来用。

**教训**：
- 知识管理 = 存储 + 索引 + 激活规则
- 每个知识卡片必须有"触发条件"
- 建立场景分类索引，遇到问题时先查索引

**应用**：
- 创建了 `memory/learning/knowledge/INDEX.md`
- 在 AGENTS.md 增加知识激活规则
- 设计了包含触发条件的知识卡片模板

### 及时记录原则（2026-03-06）
**问题**：只在会话结束时更新日志，如果异常结束会丢失数据。

**教训**：
- 心跳时就要更新日志和记忆
- 不能等到会话结束才记录
- 重要工作完成后立即写入文件

**应用**：
- 在 HEARTBEAT.md 增加"更新日志"和"更新记忆"任务
- 每次心跳都记录工作到 daily/YYYY-MM-DD.md

## 工具使用

### 主动识别和使用工具（2026-03-06）
**原则**：泽钢不会明确说"用某个 skill"，我要自己判断并使用。

**教训**：
- 根据任务类型主动选择合适的工具
- 网页抓取 → web-scraping skill
- 数据处理 → Codex
- 浏览器交互 → agent-browser skill

**应用**：
- 为嘟嘟巴士项目配置了完整工具链
- 不需要等明确指示，主动使用工具

### RICE 框架用于优先级决策（2026-03-07）
**场景**：有多个待办任务，不知道先做哪个。

**工具**：product-manager-toolkit 的 RICE 优先级框架

**教训**：
- 用数据驱动决策，而不是凭感觉
- RICE = (Reach × Impact × Confidence) / Effort
- Quick Win（高价值低成本）优先于个人学习任务

**应用**：
- 分析了5个待办任务，发现"测试prompt生成质量"ROI最高
- 个人学习任务优先级最低（只影响1人）
- 建立了量化评估习惯
