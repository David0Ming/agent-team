# 知识卡片：竞态条件 (Race Condition)

## 核心概念
多个操作同时访问共享资源，结果取决于执行顺序

## 问题示例
```
线程A: 读取 count=5
线程B: 读取 count=5
线程A: count+1 = 6，写回
线程B: count+1 = 6，写回
结果: count=6（应该是7！）
```

## 解决方案

### 1. 锁 (Lock)
- 同一时间只能一个操作
- 悲观锁/乐观锁

### 2. 原子操作
- 不可中断的操作
- 如：atomic++

### 3. 事务
- 一组操作原子执行
- 全部成功或全部失败

## 应用场景
- 并发配置修改
- 状态更新保护
- 资源竞争

## 我们可以用在哪
- 多请求同时修改配置
- 并发 API 调用保护
- 状态一致性
