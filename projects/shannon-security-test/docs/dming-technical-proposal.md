## 三、技术方案框架

基于 Shannon 的要求，提供通用实施框架：

### 3.1 环境选择建议

**推荐方案**: 优先使用 Staging 环境

| 环境 | 优势 | 劣势 | 推荐度 |
|------|------|------|--------|
| **Staging** | 接近生产配置<br>数据相对真实<br>测试结果更有参考价值 | 可能影响测试团队工作<br>需要协调时间窗口 | ⭐⭐⭐⭐⭐ |
| **Dev** | 完全隔离<br>可随时测试<br>无协调成本 | 配置可能不完整<br>数据不真实<br>结果参考价值低 | ⭐⭐⭐ |

**决策建议**:
- 如果 Staging 环境稳定且可用 → 选择 Staging
- 如果 Staging 环境不存在或不稳定 → 先搭建独立测试环境
- 避免在 Dev 环境测试（结果可信度低）

### 3.2 源代码准备方案

**标准流程**:
```bash
# 1. 克隆目标仓库
git clone <repository-url>
cd <project-name>

# 2. 切换到对应分支
git checkout staging  # 或 develop

# 3. 确认代码完整性
ls -la
cat package.json  # 或其他依赖文件

# 4. 打包为 Shannon 可用格式
tar -czf source-code.tar.gz .
```

**注意事项**:
- 确保包含所有依赖配置文件
- 移除敏感信息（.env 文件需要单独处理）
- 确认代码版本与测试环境一致

### 3.3 Docker 配置方案

**前置条件检查**:
```bash
# 检查 Docker 是否安装
docker --version
docker-compose --version

# 检查 Docker 运行状态
docker ps
```

**Shannon 运行环境准备**:
```bash
# 1. 拉取 Shannon 镜像（假设）
docker pull shannon-ai/penetration-test:latest

# 2. 准备配置文件
cat > shannon-config.yml <<EOF
target:
  source_code: /app/source
  environment: staging
  api_endpoint: https://staging.dudubus.com
anthropic:
  api_key: \${ANTHROPIC_API_KEY}
test_options:
  destructive: true
  time_limit: 90m
EOF

# 3. 运行测试
docker run -v $(pwd):/app/source \
  -e ANTHROPIC_API_KEY=sk-xxx \
  shannon-ai/penetration-test:latest
```

**资源需求**:
- CPU: 2核以上
- 内存: 4GB 以上
- 磁盘: 10GB 可用空间
- 网络: 稳定的外网访问（调用 Anthropic API）

### 3.4 Anthropic API Key 配置

**获取方式**:
1. 访问 https://console.anthropic.com
2. 创建 API Key（如果没有）
3. 设置使用限额（建议单次测试限额 $100）

**安全配置**:
```bash
# 方式1: 环境变量（推荐）
export ANTHROPIC_API_KEY="sk-ant-xxx"

# 方式2: 配置文件（需加密存储）
echo "ANTHROPIC_API_KEY=sk-ant-xxx" > .env
chmod 600 .env

# 方式3: Docker secrets（生产级）
echo "sk-ant-xxx" | docker secret create anthropic_key -
```

**成本控制**:
- 单次测试预算: $50
- 建议设置 API 使用上限: $100
- 监控实际消耗，避免超支

## 四、技术风险评估

### 4.1 破坏性风险 ⚠️ 高危

**风险描述**: Shannon 是破坏性测试工具，可能：
- 修改数据库记录
- 删除文件或数据
- 触发业务逻辑异常
- 发送真实通知/消息

**缓解措施**:
- ✅ **强制要求**: 仅在隔离环境运行
- ✅ 测试前完整备份数据库
- ✅ 禁用外部服务调用（支付、短信、邮件）
- ✅ 设置数据库只读账号（如果可能）
- ✅ 准备快速回滚方案

### 4.2 数据泄露风险 ⚠️ 中危

**风险描述**: 
- 源代码包含敏感信息（API keys, 数据库密码）
- Shannon 需要上传代码到 Anthropic（虽然声称不存储）
- 测试报告可能包含敏感路径/配置

**缓解措施**:
- ✅ 清理源代码中的敏感信息
- ✅ 使用测试环境专用密钥（非生产密钥）
- ✅ 审查 Shannon 的数据处理协议
- ✅ 测试报告脱敏后再分享

### 4.3 环境隔离风险 ⚠️ 中危

**风险描述**:
- Staging 环境可能与生产环境有网络连接
- 测试可能触发生产环境的依赖服务
- 数据库可能有主从复制关系

**缓解措施**:
- ✅ 确认网络隔离（防火墙规则）
- ✅ 断开与生产环境的所有连接
- ✅ 使用独立的第三方服务账号（支付/短信等）
- ✅ 检查数据库复制配置

### 4.4 时间窗口风险 ⚠️ 低危

**风险描述**:
- 测试时间 1-1.5 小时，可能超出 11:00-17:00 窗口
- 测试期间环境不可用，影响其他团队

**缓解措施**:
- ✅ 选择周中进行（避开周一/周五）
- ✅ 提前通知相关团队
- ✅ 预留缓冲时间（12:00-15:00 执行）

### 4.5 成本超支风险 ⚠️ 低危

**风险描述**:
- API 调用可能超出 $50 预算
- 测试失败需要重跑

**缓解措施**:
- ✅ 设置 API 使用上限
- ✅ 监控实时消耗
- ✅ 准备 $150 预算（3次测试）
