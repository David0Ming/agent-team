# 技能使用规则 - 主动判断何时使用

## 核心原则
**不要等明确指示，主动判断并使用技能！**

## 工作流检查点

### 1. 开始任务前
- [ ] **token-estimator** - 大任务（>1000行代码）先估算成本
- [ ] **find-skills** - 需要新功能时，先搜索是否有现成技能

### 2. 编写代码时
- [ ] **test-driven-development** - 实现功能前先写测试
- [ ] **systematic-debugging** - 遇到bug时系统化调试

### 3. 完成代码后
- [ ] **git-commit** - 提交代码时生成规范的commit message
- [ ] **requesting-code-review** - 完成任务后请求审查

### 4. 收到反馈时
- [ ] **receiving-code-review** - 收到代码审查反馈时验证

### 5. 需要信息时
- [ ] **tavily-search** - 需要最新信息时搜索
- [ ] **last30days** - 研究最近30天趋势
- [ ] **web-scraping** - 需要抓取网页数据

### 6. 浏览器操作时
- [ ] **agent-browser** - 需要登录、交互的网站
- [ ] **browser-auto** - 简单的浏览器自动化

### 7. 定期维护
- [ ] **self-updater** - 每周检查OpenClaw更新
- [ ] **weather** - 用户问天气时

### 8. 产品决策时
- [ ] **product-manager-toolkit** - 需要优先级分析时用RICE

## 自我检查
每次完成任务后问自己：
1. 我应该用哪些技能但没用？
2. 为什么没用？
3. 下次如何改进？

## 记住
**技能是工具，不是装饰品。不用就是浪费！**
