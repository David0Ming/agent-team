# 技能索引 - 按场景分类

> 💡 **使用说明**：遇到相关场景时，查阅对应技能的 SKILL.md
> 📊 **当前技能数量**：15个（workspace: 8个 + openclaw: 7个）

## 💻 开发工作流

**何时使用**：编写代码、提交、审查时

- `git-commit` - Git提交规范，conventional commit分析和生成 (📍 ~/.openclaw/skills/)
- `test-driven-development` - TDD实践，先写测试再实现 (📍 ~/.openclaw/skills/)
- `requesting-code-review` - 完成任务后请求代码审查 (📍 ~/.openclaw/skills/)
- `receiving-code-review` - 接收代码审查反馈，技术验证 (📍 ~/.openclaw/skills/)

## 🐛 调试与测试

**何时使用**：遇到bug、测试失败、需要调试时

- `systematic-debugging` - 系统化调试方法，bug分析 (📍 ~/.openclaw/skills/)
- `webapp-testing` - Web应用测试，Playwright交互 (📍 ~/.openclaw/skills/)

## 🌐 Web 自动化与爬虫

**何时使用**：需要浏览器自动化、网页交互、数据抓取时

- `agent-browser` - 浏览器自动化，导航、快照、交互、截图 (📍 workspace/skills/)
- `web-scraping` - Web爬虫，反爬虫绕过，内容提取 (📍 ~/.openclaw/skills/)

## 🔍 搜索与研究

**何时使用**：需要搜索信息、研究主题时

- `tavily-search` - Tavily 搜索 API (📍 workspace/skills/)
- `tavily-ai-skills` - Tavily AI 搜索技能 (📍 workspace/skills/)
- `last30days` - 最近30天的 Reddit/X/Web 研究 (📍 workspace/skills/)

## 🛠️ 技能管理

**何时使用**：需要发现、安装新技能时

- `find-skills` - 从 skills.sh 发现和安装新技能 (📍 workspace/skills/)

## 📊 产品管理

**何时使用**：产品决策、优先级分析、PRD编写时

- `product-manager-toolkit` - RICE分析、PRD模板、GTM策略 (📍 workspace/skills/)

## ⚙️ 系统维护

**何时使用**：系统更新、维护时

- `self-updater` - OpenClaw 自动更新 (📍 workspace/skills/)

## 📈 性能优化

**何时使用**：大型任务、需要估算 token 使用时

- `token-estimator` - Token 估算，模型选择 (📍 workspace/skills/)

## 🎯 使用提醒

**写代码前** → `test-driven-development`
**遇到bug时** → `systematic-debugging`
**提交代码时** → `git-commit`
**完成任务后** → `requesting-code-review`
**收到反馈时** → `receiving-code-review`
**需要爬虫时** → `web-scraping`
**需要浏览器操作** → `agent-browser`
**需要搜索** → `tavily-search` 或 `last30days`
**产品决策** → `product-manager-toolkit`
**大型任务前** → `token-estimator`

- 新增技能时，必须明确使用场景
- 保持技能数量精简，避免冗余
